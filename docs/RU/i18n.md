# Локализация ovlSysmodules (Ryazhenka)

Этот файл описывает как работает русско-английская локализация форка
и как добавлять новые языки.

> Связано: общая идентичность форка (подпись `RYZH` вместо `ULTR`,
> swap submodule, брендинг) — см. [`../../README.md`](../../README.md)
> и [`../../CHANGELOG.md`](../../CHANGELOG.md).

## Архитектура

```
source/main.cpp
  └─ initServices()
       ├─ setInitialize()
       ├─ setGetSystemLanguage(&langCode)   ← языковой код системы
       ├─ setMakeLanguage(langCode, &lang)  ← SetLanguage enum
       └─ ovls::loadLanguage("ru" | "en")
                │
                ▼
source/ovls_lang.cpp
  └─ loadFile("sdmc:/config/ryazhahand/ovlSysmodules/lang/<code>.json")
       ├─ cJSON_ParseWithLength()
       └─ для каждого ключа из kTable: *e.dst = item->valuestring
```

Дефолтные английские значения скомпилированы прямо в бинарник
(`source/ovls_lang.cpp`), так что если JSON-файл отсутствует или
повреждён — оверлей не упадёт, а покажет upstream-defaults.

## Путь установки

JSON-файлы лежат на SD-карте по пути:

```
/config/ryazhahand/ovlSysmodules/lang/en.json
/config/ryazhahand/ovlSysmodules/lang/ru.json
```

Релизный zip распаковывается в корень SD-карты и кладёт их туда
автоматически.

## Маппинг языка системы → код

Реализован в `source/main.cpp::initServices`:

| HOS SetLanguage | 2-letter code | JSON-файл |
|-----------------|---------------|-----------|
| `SetLanguage_Russian` (10) | `ru` | `lang/ru.json` |
| любой другой | `en` | `lang/en.json` |

Добавление нового языка = новый `case` в switch + новый `lang/<code>.json`.

## Switch system-font glyphs

В upstream `DYNAMIC_HEADER`/`STATIC_HEADER` встроены button-glyph иконки
из PUA (Private Use Area) Switch system font:

| Codepoint | Что | Где используется |
|-----------|-----|-------------------|
| `U+E016` | info-иконка | префикс хинтов |
| `U+E031` | small separator dot | разделитель в headers |
| `U+E0E0` | A button glyph | "Toggle" подсказка |
| `U+E0E3` | Y button glyph | "Auto Start" подсказка |
| `U+E098` | "off"/cleared индикатор | `ICON_NO_FLAG` префикс On/Off значений |
| `U+E0F4` | "on"/check индикатор | `ICON_FLAG` префикс On/Off значений |
| `U+E150` | error icon | большая иконка на "No sysmodules found!" экране |

В JSON-файлах эти символы записываются как UTF-8 escape-последовательности.
Например в `en.json`:

```json
"DYNAMIC_HEADER": "Dynamic   Auto Start   Toggle"
```

После загрузки в C++ они корректно рендерятся Tesla renderer'ом как
иконки на консоли.

## Иконки `On`/`Off`

`statusDescription()` в `source/gui_main.cpp` строит строку из двух частей:

```cpp
buf = (hasFlag ? ovls::ICON_FLAG : ovls::ICON_NO_FLAG)  // префикс-индикатор автозапуска
    + (running ? ovls::ON_LABEL  : ovls::OFF_LABEL );   // локализованный On/Off
```

То есть префикс показывает есть ли `boot2.flag` (галочка vs зачёркнутый
кружок), а суффикс — текущее состояние процесса. 4 варианта:

| running | hasFlag | Что показано |
|---------|---------|--------------|
| false | false | ` Off` (модуль не запущен, автозапуск выкл) |
| false | true | ` Off` (модуль не запущен, но автозапуск вкл — поднимется после reboot) |
| true | false | ` On` (модуль запущен, но не на автозапуске) |
| true | true | ` On` (модуль запущен и на автозапуске — норма) |

## Что добавить если новый key понадобится

1. `include/ovls_lang.hpp` — `extern std::string NEW_KEY;`
2. `source/ovls_lang.cpp` — `std::string NEW_KEY = "default English";` + строка в `kTable`.
3. `lang/en.json`, `lang/ru.json`, и все остальные — добавить ключ.
4. В `source/gui_main.cpp` (или где нужно) — `ovls::NEW_KEY.c_str()`.
